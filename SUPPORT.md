## 2. 提问指南

### 2.1 提问前

1. **搜索**：在 Issue 和 Discussion 中搜索是否已有答案
2. **阅读文档**：查阅项目 README、Wiki 和 FAQ
3. **尝试复现**：确保你理解问题的具体表现

### 2.2 好的提问方式

```
✅ "我在使用 ITrys CLI v1.2.0 生成 SBOM 时，在 Windows 11 上遇到以下错误……"
✅ "我阅读了文档中关于……的部分，但不确定如何处理……的情况"
✅ "以下是我的复现步骤和完整错误日志……"
```

### 2.3 不好的提问方式

```
❌ "它不工作了"（缺少上下文）
❌ "帮我看看这个"（没有具体描述）
❌ "这个东西太难用了"（没有建设性反馈）
```

### 2.4 提问模板

```markdown
## 环境
- 项目及版本：
- 操作系统：
- 相关依赖版本：

## 问题描述
我想要实现……，但遇到了……

## 已尝试
1. 我已经阅读了……
2. 我尝试了……
3. 我搜索了……但没有找到……

## 错误信息
```
粘贴完整的错误日志
```

## 补充信息
截图、配置文件等
```

## 3. 社区互助

### 3.1 帮助他人

- **回答问题**：在 Discussion 或 Discord 中帮助他人解决问题
- **完善文档**：如果你找到了解决方案，欢迎提交 PR 更新文档
- **分享经验**：在 Discussion 中分享你的使用心得

### 3.2 成为维护者

如果你持续贡献并希望深度参与，可以申请成为项目维护者：

1. 持续贡献 ≥3 个月
2. 至少有 5 个被合并的 PR
3. 由现有维护者提名
4. PMC 确认

详见 [GOVERNANCE.md](./GOVERNANCE.md#22-角色定义)

## 4. 常见问题（FAQ）

### 通用

**Q: ITrys 是什么？**
A: ITrys（艾雀斯）是一个开放、中立、可持续的开源协作组织，致力于打造企业级应用开发。

**Q: 加入 ITrys 需要付费吗？**
A: 不需要。加入 ITrys 完全免费，我们提供零成本接入。

**Q: 加入后可以退出吗？**
A: 可以。只需在 community 仓库提交一个 Issue 即可退出，无任何绑定。

### 项目入驻

**Q: 什么样的项目可以入驻？**
A: 使用 OSI 批准的开源许可证、可独立构建运行、功能完整（非演示版）的项目。详见 [GOVERNANCE.md](./GOVERNANCE.md#41-基本要求)。

**Q: 商业项目可以入驻吗？**
A: 可以。ITrys 支持多种合法商业模式（Open Core、SaaS 等），但须明确声明边界且开源版须功能完整。

**Q: 派生项目可以入驻吗？**
A: 可以，但须已完成"去 fork 化"并包含显著的原创性增强，且在 README 中致谢原项目。

**Q: 入驻审核需要多久？**
A: PMC 初审 7 天内完成，社区讨论期 ≥7 天，整体流程通常 2-4 周。

### 贡献

**Q: 我是新手，如何开始贡献？**
A: 从标有 `good-first-issue` 的 Issue 开始，详见 [CONTRIBUTING.md](./CONTRIBUTING.md)。

**Q: 必须签署 CLA 吗？**
A: 不需要。ITrys 使用 DCO（开发者原创证书），只需在提交时 `git commit -s` 即可。

**Q: 贡献的代码版权归谁？**
A: 贡献者保留其贡献的版权。ITrys 不主张任何知识产权。

### 安全

**Q: 如何报告安全漏洞？**
A: 请发送邮件至 security@itrys.org，**不要**在公开 Issue 中报告。详见 [SECURITY.md](./SECURITY.md)。
