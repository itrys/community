# ITrs 认证徽章标准与审计流程（Certification Program）

> **徽章是信任的符号，认证是合规的证明。**

---

## 1. 概述

ITrys 提供两种项目认证徽章，代表不同阶段的合规状态：

| 徽章 | 名称 | 含义 | 获取条件 |
|------|------|------|----------|
| 🌱 | **ITrys Incubating** | 孵化项目 | 通过入驻初审 |
| 🏅 | **ITrys Certified** | 认证项目 | 通过完整合规审计 |

> ⚠️ 未通过审核的项目**不得**使用任何 ITrs 认证徽章。违规使用将按 [TRADEMARK.md](./TRADEMARK.md) 处理。

---

## 2. 认证标准

### 2.1 Incubating（孵化）徽章标准

项目须**同时满足**以下全部条件：

| # | 标准项 | 具体要求 | 验证方式 |
|---|--------|----------|----------|
| 1 | **开源许可证** | 使用 OSI 批准的开源许可证 | 检查仓库 LICENSE 文件 |
| 2 | **可独立构建** | 提供 README / 构建文档，CI 能通过 | 实际执行构建 |
| 3 | **功能完整** | 核心功能可用，非演示版/阉割版 | 功能测试或演示验证 |
| 4 | **依赖合规** | 所有直接依赖许可证与项目许可证兼容 | 自动化扫描 + 人工确认 |
| 5 | **DCO 签署** | 所有合并的提交包含 `Signed-off-by` | DCO 检查机器人验证 |
| 6 | **维护承诺** | 至少 1 名维护者承诺 6 个月内持续维护 | 入驻申请中书面承诺 |
| 7 | **SBOM 生成** | 首次生成 SBOM（SPDX 格式） | 检查 compliance-reports 目录 |
| 8 | **行为准则** | 项目须遵守 ITrs 社区行为准则 | 入驻申请中确认 |

### 2.2 Certified（认证）徽章标准

在满足 Incubating 全部标准的基础上，还须**同时满足**以下条件：

| # | 标准项 | 具体要求 | 验证方式 |
|---|--------|----------|----------|
| 1 | **孵化期满** | 孵化期 ≥3 个月且状态正常 | 检查入驻时间记录 |
| 2 | **活跃维护** | 孵化期内每月 ≥1 次有效提交 | Git 提交记录统计 |
| 3 | **SBOM 更新** | 每次版本发布均更新 SBOM | 检查 compliance-reports 目录 |
| 4 | **许可证扫描** | 最近季度内无高/中风险依赖 | 许可证扫描报告 |
| 5 | **安全扫描** | 最近季度内无未修复的严重/高危漏洞 | 安全扫描报告 |
| 6 | **社区响应** | Issue 平均首次回复时间 ≤14 天 | Issue 统计 |
| 7 | **文档完备** | README + 使用文档 + API 文档（如适用） | 人工审查 |
| 8 | **商业合规**（如适用） | 开源/付费边界清晰，功能对比表完整 | 人工审查 |
| 9 | **年度自查** | 完成年度合规自查表 | PMC 确认 |

### 2.3 标准对比一览

| 标准项 | Incubating 🌱 | Certified 🏅 |
|--------|:---:|:---:|
| OSI 许可证 | ✅ | ✅ |
| 可独立构建 | ✅ | ✅ |
| 功能完整 | ✅ | ✅ |
| 依赖合规 | ✅ | ✅ |
| DCO 签署 | ✅ | ✅ |
| 维护承诺 | ≥6 个月 | 持续维护 |
| SBOM 生成 | 首次生成 | 每次版本更新 |
| 孵化期满 | — | ≥3 个月 |
| 活跃维护 | — | 每月 ≥1 次提交 |
| 许可证扫描 | — | 季度扫描无中高风险 |
| 安全扫描 | — | 季度扫描无严重高危 |
| 社区响应 | — | ≤14 天 |
| 文档完备 | — | ✅ |
| 商业合规 | — | ✅（如适用） |
| 年度自查 | — | ✅ |

## 3. 审计流程

### 3.1 Incubating 徽章审计

```
提交申请 → 自动检查（1 天）→ PMC 初审（7 天）→ 授予徽章 → 进入孵化期
```

**详细步骤**：

1. **提交申请**：项目方通过 Issue 模板提交入驻申请
2. **自动检查**（1 天内）：
   - DCO 机器人验证提交签名
   - 许可证扫描工具检查依赖
   - CI 构建验证
3. **PMC 初审**（7 天内）：
   - 审核申请材料完整性
   - 确认功能可用性（演示/测试）
   - 评估商业模式合规性（如适用）
   - 投票：简单多数通过
4. **授予徽章**：
   - 在 `awesome-itrys/README.md` 中添加孵化项目条目
   - 项目方在 README 中添加 `ITrys Incubating` 徽章
   - 发布首次 SBOM 至 `compliance-reports/`

### 3.2 Certified 徽章审计

```
孵化期满 → 项目方申请 → 自动扫描（3 天）→ PMC 审计（14 天）→ 指导委员会终审 → 授予徽章
```

**详细步骤**：

1. **申请条件**：孵化期满 3 个月，项目方主动申请或 PMC 主动发起
2. **自动扫描**（3 天内）：
   - 运行完整许可证扫描 → 生成许可证报告
   - 运行安全扫描 → 生成安全报告
   - 检查 DCO 合规率
   - 统计 Issue 响应时间
   - 生成最新 SBOM
3. **PMC 审计**（14 天内）：
   - 审查所有自动扫描报告
   - 验证文档完备性
   - 验证商业合规性（如适用）
   - 评估社区健康度
   - 产出综合合规报告
4. **指导委员会终审**：
   - 审阅综合合规报告
   - 投票：简单多数通过
5. **授予徽章**：
   - 在 `awesome-itrys/README.md` 中升级为认证项目
   - 项目方将徽章从 `ITrys Incubating` 更新为 `ITrys Certified`
   - 发布综合合规报告至 `compliance-reports/`

### 3.3 年度复审

Certified 徽章**有效期 1 年**，须通过年度复审：

| 步骤 | 时间 | 内容 |
|------|------|------|
| 自动扫描 | 到期前 30 天 | 触发季度扫描流程 |
| PMC 复审 | 到期前 14 天 | 审查年度合规自查表 + 扫描报告 |
| 结果通知 | 到期前 7 天 | 通知续期/整改/降级 |

**复审结果**：

| 结果 | 处理 |
|------|------|
| ✅ **续期** | 徽章有效期延长 1 年 |
| 🔄 **整改** | 给予 30 天整改期，期间保留徽章 |
| ⬇️ **降级** | 降为 Incubating 或归档 |

## 4. 自动化审计工具

### 4.1 自动检查项

| 检查项 | 工具 | 触发条件 | 结果 |
|--------|------|----------|------|
| DCO 签署验证 | DCO Bot | 每次 PR | 通过/阻止合并 |
| 许可证扫描 | FOSSA / ScanCode | 每月 / 每次 PR | 许可证报告 |
| 安全漏洞扫描 | Snyk / Trivy | 每次 Push | 安全报告 |
| 密钥泄露检测 | GitGuardian / gitleaks | 每次 Push | 告警 |
| SBOM 生成 | Syft / sbom-tool | 每次版本发布 | SPDX JSON |
| 依赖更新 | Dependabot / Renovate | 每周 | 自动 PR |

### 4.2 检查结果阈值

| 检查项 | Incubating 通过阈值 | Certified 通过阈值 |
|--------|---------------------|---------------------|
| DCO 合规率 | ≥90% | 100% |
| 高风险依赖 | ≤3 且有整改计划 | 0 |
| 中风险依赖 | — | 有风险评估说明 |
| 严重漏洞 | — | 0（已修复） |
| 高危漏洞 | — | 0（已修复） |
| Issue 首次回复 | — | ≤14 天（中位数） |

## 5. 徽章使用规范

### 5.1 徽章样式

```markdown
<!-- Incubating 徽章 -->
[![ITrys Incubating](https://img.shields.io/badge/ITrys-Incubating-green)](https://itrys.org)

<!-- Certified 徽章 -->
[![ITrys Certified](https://img.shields.io/badge/ITrys-Certified-blue)](https://itrys.org)
```

> 📌 徽章图片将在组织正式运营后发布于 itrys.org/brand。

### 5.2 使用位置

| 位置 | 是否必须 | 说明 |
|------|----------|------|
| 项目 README 顶部 | ✅ 必须 | 紧邻项目标题下方 |
| 项目官网首页 | 推荐展示 | 置于页脚或侧栏 |
| 项目文档站点 | 推荐展示 | 首页或关于页 |
| npm / PyPI 等包描述 | 可选 | 包描述文本中 |

### 5.3 使用要求

- ✅ 徽章须链接至 itrys.org 或项目合规报告页
- ✅ 徽章须保持原始样式，不得修改颜色或文字
- ❌ 不得在退出 ITrs 后继续使用徽章
- ❌ 不得将 Incubating 徽章标注为 Certified
- ❌ 不得在徽章过期后继续展示

### 5.4 降级与移除

| 场景 | 处理 |
|------|------|
| 项目归档 | 移除所有 ITrs 徽章 |
| 复审未通过且未整改 | Certified → Incubating |
| 连续 6 个月无维护 | Incubating → 移除徽章 |
| 项目退出 ITrs | 立即移除所有徽章 |
| 违规使用商标 | 移除徽章 + 按 TRADEMARK.md 处理 |

## 6. 审计报告

### 6.1 报告存放

所有审计报告存放于 `compliance-reports/projects/{project-name}/` 目录：

| 文件 | 说明 | 模板 |
|------|------|------|
| `{project}-sbom-v{ver}.json` | SBOM 报告 | [模板](./compliance-reports/templates/sbom-template.json) |
| `{project}-license-{date}.md` | 许可证扫描报告 | [模板](./compliance-reports/templates/license-report-template.md) |
| `{project}-report-{date}.md` | 综合合规报告 | [模板](./compliance-reports/templates/compliance-report-template.md) |

### 6.2 报告公开

- 所有合规报告**默认公开**，存放于 community 仓库
- 如涉及商业敏感信息，项目方可申请**部分脱敏**（须 PMC 批准）
- 脱敏仅限商业客户信息，合规结论不得隐藏

## 7. 争议与申诉

### 7.1 审计结果争议

如项目方对审计结果有异议：

1. 在审计报告 Issue 中提出异议并附理由
2. PMC 在 7 天内复核
3. 如 PMC 维持原判，可向指导委员会申诉
4. 指导委员会裁决为最终结果

### 7.2 徽章撤销争议

如对徽章撤销决定有异议：

1. 在 `community` 仓库提交申诉 Issue
2. 指导委员会在 14 天内审议
3. 审议期间徽章状态维持不变
