# ITrys 安全策略（Security Policy）

> **安全是信任的底线，透明是响应的保障。**

---

## 1. 支持的版本

ITrys 组织下各项目的安全更新范围如下：

| 项目状态 | 版本支持 | 安全更新 |
|----------|----------|----------|
| 活跃（Active） | 最近 2 个主要版本 | ✅ 接收并修复 |
| 孵化（Incubating） | 最新版本 | ✅ 接收并修复 |
| 归档（Archived） | 无 | ❌ 不再修复 |

各项目具体版本支持政策以项目仓库内 SECURITY.md 为准。

## 2. 报告安全漏洞

### 2.1 报告方式

**请勿在公开 Issue、PR 或讨论区报告安全漏洞。**

| 渠道 | 说明 |
|------|------|
| 📧 **security@itrys.org** | 优先推荐，加密通信 |
| 🔐 **GPG 加密** | 使用下方公钥加密敏感信息 |
| 💬 **私密渠道** | 联系项目维护者或指导委员会成员 |

### 2.3 报告内容

一份好的安全报告应包含：

```markdown
## 漏洞描述
简要描述漏洞类型和影响

## 影响范围
- 受影响的项目和版本
- 攻击场景和潜在危害

## 复现步骤
1. ...
2. ...

## 修复建议（选填）
你认为可能的修复方案

## 报告者信息
- 姓名/昵称（选填）
- 联系方式
- 是否希望公开致谢
```

## 3. 响应流程

### 3.1 时间线

| 阶段 | 时间目标 | 说明 |
|------|----------|------|
| **确认接收** | 24 小时内 | 确认已收到报告 |
| **初步评估** | 3 个工作日内 | 评估漏洞严重性与影响范围 |
| **修复开发** | 视严重度而定 | 详见下方 SLA |
| **发布修复** | 修复完成后 72 小时内 | 发布安全版本 |
| **公开披露** | 修复发布后 30 天内 | 发布安全公告 |

### 3.2 严重度与 SLA

| 严重度 | 定义 | 修复 SLA | 示例 |
|--------|------|----------|------|
| 🔴 **严重（Critical）** | 可被远程利用，影响数据安全或系统可用性 | 7 天内 | RCE、SQL 注入、认证绕过 |
| 🟠 **高危（High）** | 可被利用但有限制条件 | 14 天内 | XSS、CSRF、权限提升 |
| 🟡 **中危（Medium）** | 影响有限或利用难度较高 | 30 天内 | 信息泄露、DoS |
| 🟢 **低危（Low）** | 影响极小 | 下个版本 | 配置建议、最佳实践 |

## 4. 披露政策

### 4.1 协调披露

ITrys 遵循**协调披露（Coordinated Disclosure）**原则：

1. 报告者私下报告漏洞
2. ITrys 在修复期内保密漏洞详情
3. 修复发布后，双方协调公开披露时间
4. 默认在修复发布后 **30 天**公开披露

### 4.2 报告者权益

- **致谢**：在安全公告中致谢报告者（除非要求匿名）
- **CVE**：协助申请 CVE 编号
- **赏金**：ITrys 当前无官方赏金计划，但不阻止项目方自行设立

### 4.3 例外情况

以下情况 ITrys 可能提前公开披露：

- 漏洞已被公开利用（0-day）
- 报告者自行公开披露
- 超过 90 天未收到报告者回复且修复已完成

## 5. 安全最佳实践

### 5.1 对项目维护者

- 定期更新依赖项，修复已知漏洞
- 启用依赖项自动安全更新（Dependabot / Renovate）
- 在 CI 中集成安全扫描（SAST / DAST / SCA）
- 遵循最小权限原则配置 CI/CD
- 不在代码中硬编码密钥或凭证

### 5.2 对贡献者

- 不提交含有硬编码密钥的代码
- 使用环境变量或密钥管理服务
- 发现安全问题时通过私密渠道报告
- 遵循项目的安全编码规范

## 6. 安全审计

### 6.1 自动化检查

ITrys 为入驻项目提供以下自动化安全检查：

| 检查项 | 工具 | 频率 |
|--------|------|------|
| 依赖漏洞扫描 | SCA 工具 | 每次构建 |
| 许可证合规扫描 | SBOM 生成 | 每月 |
| 代码安全扫描 | SAST 工具 | 每次 PR |
| 密钥泄露检测 | Secret Scanning | 每次 Push |

### 6.2 合规报告

- 安全扫描结果纳入项目合规报告
- 发布至 [compliance-reports](https://gitee.com/ITrys/compliance-reports)
- 严重漏洞修复记录公开可查
