## 🌱 关于我们

**ITrys（发音：/aɪˈtriːs/，中文名：艾雀斯）** **是开放、中立、可持续**的开源协作组织，致力于打造企业级应用开发。

"ITrys" 源于 "**I Try**"——  
我们相信，每一次尝试都是创新的起点。  
每一个微小的尝试都值得被鼓励，每一份真诚的分享都值得被看见。

无论你是初学者还是资深专家，只要愿意动手、乐于共享，这里就是你的舞台。  
艾雀斯的本质，就是将无数个 "I Try" 汇聚成改变世界的 "**We Did**"。

---

### 🌟 我们的使命

- ✅ **降低参与门槛**：帮助每个人从"用开源"走向"做开源"，让第一个 PR 不再令人却步；
- ✅ **孵化实用开源**：聚焦高质量、可复用的开发框架、工具与基础设施，直击开发者真实痛点。
- ✅ **共建中文生态**：营造友好、包容、互助的社区文化，让共享成为习惯，共建成为常态，推动本土开源可持续发展。

> 💡 新手友好 · 问题无大小 · 回答有温度

## 🎯 核心理念

- **开放透明**：代码、讨论、决策，全部公开。

- **务实不炫技**：能跑起来、有人用，比"技术先进"更重要。

## 🛠️ 商业友好（Business-Friendly）

- ITrys 明确支持以下**合法且可持续**的开源商业模式：

| 模式 | 说明 | 示例 |
|------|------|------|
| **Open Core** | 开源基础功能 + 闭源高级特性 | GitLab CE / EE |
| **托管服务（SaaS）** | 提供托管运维与 SLA 支持 | MongoDB Atlas |
| **定制开发** | 接受企业付费定制需求 | 个人开发者接私单 |
| **教育/科研开源** | 高校项目长期托管与交接 | 实验室毕业项目 |
| **双许可（Dual License）** | 仅限 OSI 许可证 + 商业授权（需明确声明） |  |

> 🔒 **原则**：开源部分必须**可构建、可运行、功能完整**，非"演示版"。

- **禁止行为**：
  - ❌ 开源版故意无法使用（如禁用关键 API）
  - ❌ 模糊商业边界（如"高级版更稳定"但无证据）

### 🔹 我们郑重承诺：

> ✅ **"ITrys 不拥有你的代码，版权始终归你所有。"**

> ✅ **"你的项目仍是独立品牌，ITrys 仅提供基础设施支持。"**

> 没有合同、没有锁定期、没有隐藏条款。

## 🚀 如何加入？

1. **确保项目符合 [准入标准](./GOVERNANCE.md#4-项目准入标准)**
   - 使用 MIT / Apache-2.0 许可证
   - 可独立构建运行
   - 接受社区合理贡献（如文档、bug fix）

2. **提交申请**
   → [点击创建「项目入驻申请」Issue](https://gitee.com/ITrys/community/issues/new?template=project_proposal.yml)

3. **通过审核后**
   - 获得 `ITrys Certified` 徽章
   - 报告发布至 [compliance-reports](./compliance-reports/)
   - 项目收录至 [awesome-itrys](./awesome-itrys/)

### 💡 关于派生项目（Forked Projects）
- ITrys 接受对现有开源项目的增强型派生项目申请。
- 申请前，项目必须：
    - 已完成 "去 fork 化"（在托管平台显示为独立仓库）
    - 包含显著的原创性增强（如新功能、国产化适配、性能优化）
    - 在 README 中致谢原项目（体现开源精神）
    - ITrys 不接受仅做微小修改或未活跃维护的派生项目。
