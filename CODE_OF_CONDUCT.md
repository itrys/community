# ITrys 社区行为准则（Code of Conduct）

> **技术以解决问题为本，协作以相互尊重为先。**

---

## 1. 我们的承诺

作为 ITrys 社区成员，我们承诺让每位参与者都能在免受骚扰和歧视的环境中协作，无论其：

- 年龄、体型、残疾状况
- 种族、国籍、文化背景
- 性别、性别认同与表达
- 性取向
- 宗教信仰或无宗教信仰
- 技术水平或经验深浅
- 教育程度或社会地位
- 语言能力

我们承诺以专业、尊重的方式互动，**再简单的问题，也值得认真回答。**

---

## 2. 我们的标准

### ✅ 鼓励的行为

- **友善包容**：欢迎不同背景的参与者，尊重不同观点
- **耐心指导**：对新手问题保持耐心，用鼓励代替批评
- **建设性反馈**：提出问题时附带改进建议
- **尊重隐私**：未经允许不公开他人的私人信息
- **及时回应**：对 Issue、PR 尽量在合理时间内回复
- **感恩贡献**：对他人的贡献表示感谢与认可
- **知错能改**：意识到不当行为时主动道歉并改正

### ❌ 禁止的行为

- **歧视性言论**：基于上述任何特征的侮辱或贬低
- **骚扰行为**：不受欢迎的言语或行为，包括性暗示评论、威胁、跟踪
- **人身攻击**：对个人而非观点进行攻击
- **恶意破坏**：故意提交恶意代码、垃圾信息或无意义内容
- **未经授权发布**：公开他人私密信息（doxing）
- **霸凌与排斥**：故意忽视、排挤或边缘化他人
- **学术/技术傲慢**：以"这个问题太简单了"等态度打击他人
- **滥用职权**：利用维护者权限进行不当审查或拒绝合理贡献

---

## 3. 适用范围

本行为准则适用于 ITrys 组织管辖的**所有协作场景**，包括但不限于：

- GitHub / Gitee 仓库（Issue、PR、代码评审、Wiki）
- Discord、微信群等即时通讯频道
- 邮件列表与论坛
- 线下活动与会议
- 社交媒体上代表 ITrys 的发言

## 4. 举报与处理

### 4.1 举报方式

如你经历或目睹违反行为准则的事件，请通过以下方式举报：

| 渠道 | 适用场景 |
|------|----------|
| 📧 **conduct@itrys.org** | 所有举报（优先推荐，保密处理） |
| 📝 **私密 Issue** | 在 `community` 仓库提交 [私密 Issue](https://gitee.com/ITrys/community/issues) |
| 💬 **直接联系指导委员会成员** | 紧急情况 |

> 所有举报将**严格保密**，举报者不会因举报而受到报复。

### 4.2 处理流程

```
举报 → 初步评估（3 个工作日内）→ 调查（7 个工作日内）→ 裁决 → 执行 → 通知
```

1. **初步评估**：确认举报内容是否属于本准则管辖范围
2. **调查**：收集相关证据（截图、链接等），必要时联系双方了解情况
3. **裁决**：根据违规严重程度做出处理决定
4. **执行**：实施处罚并记录
5. **通知**：将结果通知举报者和被举报者

### 4.3 处罚措施

| 级别 | 违规行为 | 处罚 |
|------|----------|------|
| **提醒** | 无意冒犯、轻微不当言论 | 私下提醒，要求改进 |
| **警告** | 重复轻微违规 | 公开警告 + 行为改进要求（30 天观察期） |
| **限制** | 较严重违规 | 限时禁言/禁止提交 PR（7–30 天） |
| **驱逐** | 严重歧视、骚扰、威胁、恶意行为 | 永久禁止参与所有 ITrys 项目 |

- 处罚决定由**项目管理委员会**（日常）或**指导委员会**（严重）做出
- 被处罚者可向**顾问委员会**提出申诉（14 天内）
- 申诉期间处罚照常执行
